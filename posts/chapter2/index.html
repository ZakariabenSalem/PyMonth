Functions

Let’s finish with some functions real quick

Most of the time you won’t write all the code from scratch. Nearly in any “more advanced” scripts you will need to use a library or a module , here is how you can import one :

Importing

Global import

import json

If we use this , will get all the main functions of the JSON module.

Short naming

import json as j

If we use this form , we can call the JSON functions in this form :

j.some_function()

importing specific functions

from module import function

now let’s create our own function in a seperate file and import it : Create a file named main.py and another named func.py .

Here is the code for the main.py :

import func as f

def __main__():
  f.print()

if __name__ == '__main__':
  __main__()

Before we code func.py , let’s have a look at our code above . noticed the “ __ “ ? well the name main is a conventional statement used by python developers to indicate that the function is the main one.

let’s also understand this part :

if __name__ == '__main__':
  __main__()

since python can’t call the functions on its own , we should do it manually. the variable __name__ is generated by the interpreter and appended in the first line of the script.

always keep this as the last line since upon reading it the python would jump directly to the main function. as a good rules of thumbs , keep it at the end of the file. This variable let us define a function and act like an external module. Thus our functions can be accessed by the main method and other scripts.

Files

standard open method

In order to use files in python you have to use the open method. It has for parameters the path of the file ( relative or absolute) and the mode.

Modes :
- r: reading only
- w: writing only and will override any previous data
- a: append to the end of the file
- b: reading binary
- U: for Universal ( discover it and tell us what you found )

open('path to file','r')

Reading text files

Let’s make a basic function that read the content of a test file

def readfile():
    file =open('testfile.txt','r')
    str=file.read()
    return(str)

as you can see , I saved the result of the open method in a variable so I can access it and Its methods such as read. ** NOTE : the variable file can be iterable** Here is how you can iterate it :

def loopread(file):
    for line in file :
        print(line, end='')

Here file represents the result of the open method.

file.close()

If you use “open” it makes sense to close the file afterwards , since it becomes a shared resource. However in the case of read this doesn’t affect much the course of our program. However , we’re lucky to have the with block. a combination of both would be as such :

def readwith():
    with open('testfile.txt')as f:
        return(f.read())

If we use the with block , there will be an automatic f.close() at the end of it.BUT ! any data collected inside the block would persist.

Now here is a method that takes a path of a text file and returns one line of it :

def readFromFile(path):
    with open(path,'r') as f:
        print(f.readline())

Manipulating Structured Data

In some cases , you will have to manipulate a structured data format such as a dictionary. However saving it as a text file would be a pain in the a$$ . BUT If you followed carefully in our previous chapter , we mentioned that dictionaries are similar to JSON files.

Writing to a JSON file

note : Json files require more structured data than txt files , that’s is why it is preferred you use the dict/json combo.

Here in this example we used the json module , it comes prepackaged with any minimal python installation.

import json

def writeJson(path):
    with open(path,'w') as fp:
        data={"name":"mahmoud","job":"helps people not like zak"}
        json.dump(data,fp)

Here is another variation:

def write2Json(path,data):

5. Classmethod

A class method is a method that’s shared among all objects. To call a class method, put the class as the first argument.

class blob:
    name = 'am a blob'

    @classmethod
    def printName(c):
        print(c.name)

blob.printName()

Let’s try it out :

b=blob()
b.printName()

How can it be the same value ?? Well its quite simple , the classmethod is shared between all objects of ‘blob’and the name variable is static shared one , thus any object created will print the same message

let’s try to change the name :

b=blob()
b.name="new name"
b.printName()
bloby=blob()
bloby.printName()

well by changing it through the object it will be ignored, however if we change it via the class it self we can obtain a different result :

blob.name="blob"
bloby=blob()
bloby.printName()

Pythonic notation :

blob.method=classmethod(blob.printName())
